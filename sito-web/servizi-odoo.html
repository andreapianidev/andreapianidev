Domande frequenti su Odoo

Cos'è Odoo e perché sceglierlo come ERP? ▼
Odoo è una suite ERP open source modulare che integra CRM, vendite, acquisti, magazzino, contabilità, produzione (MRP), e-commerce, HR, project management e marketing in un'unica piattaforma. Disponibile in versione Community (gratuita, open source LGPL) ed Enterprise (a pagamento). Vantaggi: oltre 40 moduli ufficiali, 30.000+ moduli community OCA, interfaccia moderna, scalabilità da micro-impresa a multinazionale, costi di licenza nettamente inferiori a SAP, Oracle, Microsoft Dynamics. Per il mercato italiano è la scelta più equilibrata in termini di rapporto funzionalità/costo/manutenibilità.
Qual è la differenza tra Odoo Community e Odoo Enterprise? ▼
Community è gratuita e include CRM, vendite, acquisti, magazzino, contabilità base, progetti, sito web. Enterprise (€31,10/utente/mese list, €24,90 con annual billing) aggiunge: contabilità completa con riconciliazione bancaria AI, app mobile native iOS/Android, Studio (no-code builder), Marketing Automation, Field Service, Sign, Documents DMS, MRP II avanzato, supporto ufficiale Odoo SA, upgrade gestiti, hosting Odoo.sh. Per PMI italiane fino a 15 utenti la Community con moduli OCA italiani (l10n_it_*) basta nel 60% dei casi. Vedi la tabella comparativa in questa pagina.
Quanto costa implementare Odoo in azienda? ▼
Da €3.000 a €50.000+ in base a complessità. Pacchetto starter (1-5 utenti, 3-5 moduli) €2.500-€5.000. PMI standard (10-30 utenti, 5-8 moduli, integrazioni base) €8.000-€20.000. Implementazione complessa (50+ utenti, MRP, multi-azienda, integrazioni custom, sviluppo moduli verticali) €25.000-€80.000. Aggiunge licenze Enterprise se necessarie (€31,10/utente/mese) e hosting (€30-€500/mese). Possibilità di pagamento dilazionato e formule SAL.
Quanto tempo serve per implementare Odoo? ▼
Quick-start Community con moduli standard 2-4 settimane. PMI tipica (5-10 moduli, import dati, formazione, 1-2 customizzazioni) 6-12 settimane. Progetto complesso con sviluppo verticale, integrazioni ERP legacy, migrazione storico multi-anno 4-8 mesi. Uso metodologia agile con sprint da 2 settimane e demo settimanali per ridurre rischi e validare requisiti progressivamente.
Posso migrare da Danea, Fatture in Cloud, Zucchetti, TeamSystem o SAP a Odoo? ▼
Sì, supporto migrazioni da SAP Business One, Microsoft Dynamics 365 BC, Zucchetti Mago/Ad Hoc Enterprise, TeamSystem Gamma Enterprise, Danea Easyfatt, Fatture in Cloud, Aruba Fatturazione, software custom legacy MSSQL/MySQL. Migrazione tipica include: mappatura schema dati, ETL con script Python custom, validazione integrità (matching totali, quadrature IVA), dry-run su staging, formazione team, go-live con parallel run 2-4 settimane. Costo €2.500-€15.000 in base a volume dati e complessità trasformazioni.
Si può fare upgrade da Odoo 14/15/16 a Odoo 17 o 18? ▼
Sì. Upgrade minor (es. 17.0 → 17.4) è quasi automatico. Upgrade major richiede: audit moduli custom esistenti, refactoring codice Python per nuove API (es. rimozione decoratori obsoleti, nuovo ORM), aggiornamento vista form/list a OWL framework, test regressione, migrazione dati con script OpenUpgrade. Costo upgrade community €1.500-€8.000. Per Enterprise con DB < 500MB il tool ufficiale upgrade.odoo.com è gratuito incluso nel contratto.
Odoo supporta la fatturazione elettronica italiana SDI? ▼
Sì. Odoo Enterprise include nativamente l'integrazione fatturazione elettronica italiana via SDI con provider certificati. Per Community c'è il modulo OCA l10n_it_fatturapa che genera XML FatturaPA conforme + integrazione SDI tramite intermediario (ACube, Aruba, FattureInCloud). Implementazione completa con piano dei conti italiano, esterometro, intrastat, split payment, reverse charge €1.500-€4.000. Conservazione sostitutiva a norma 10 anni inclusa.
Si può integrare Odoo con PrestaShop, WooCommerce o Shopify? ▼
Sì, sviluppo integrazioni bidirezionali Odoo ↔ e-commerce. Sincronizzazione catalogo prodotti, stock multi-magazzino, ordini, clienti, fatture, tracking spedizioni, varianti, listini multi-valuta. Per PrestaShop uso connettore OCA o sviluppo custom. Per WooCommerce/Shopify uso webhook + API REST con queue Redis per affidabilità.
